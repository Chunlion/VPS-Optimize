# 新 VPS 初始化与安全加固

这篇教程面向刚买 VPS 的第一次初始化。目标不是“把所有功能都跑一遍”，而是先把机器变成一个可维护、可回滚、不会因为一次 SSH 或防火墙操作就失联的基础环境。

推荐路径：

```text
cy -> 1 -> 2 -> 5 -> 6 -> 7 -> 8 -> 10 -> 15 -> 16
```

如果你是第一次使用，也可以进入：

```text
cy -> 新手向导 -> 1
```

## 适合谁

| 情况 | 是否适合 |
|---|---|
| 新买 VPS，准备部署面板、节点或网站 | 适合 |
| VPS 已经跑了生产业务，只想做轻量检查 | 适合，但不要直接执行 SSH/防火墙/内核改动 |
| 完全没有云厂商控制台或 VNC/救援入口 | 不建议做 SSH 改端口、防火墙收紧和内核操作 |
| OpenVZ 或很老的系统 | 不建议做内核和 BBR 相关操作 |

## 准备材料

| 材料 | 为什么需要 |
|---|---|
| VPS 快照 | SSH、防火墙、内核参数出问题时能整体回滚 |
| 当前 SSH 会话 | 改 SSH 端口后不要关闭当前窗口，先开新窗口测试 |
| 云厂商安全组权限 | 系统防火墙放行不等于云安全组放行 |
| VNC/救援控制台 | SSH 失联时最后的恢复入口 |
| SSH 公钥 | 后续建议逐步切换到密钥登录 |
| 服务器公网 IP | 验证 DNS、安全组和访问路径 |

## 预计耗时

| 步骤 | 预计耗时 |
|---|---|
| 预检 | 1-3 分钟 |
| 基础初始化 | 3-10 分钟 |
| SSH、公钥、Fail2ban | 5-10 分钟 |
| 防火墙 | 3-8 分钟 |
| 网络/内核优化 | 5-20 分钟，视所选功能而定 |
| 备份 | 1-3 分钟 |

## 会修改哪些东西

| 项目 | 可能修改内容 | 风险 |
|---|---|---|
| 系统软件包 | 常用工具、证书组件、网络工具 | 软件源异常会导致安装失败 |
| SSH | 端口、公钥认证、服务重启 | 云安全组未放行会导致失联 |
| Fail2ban | `/etc/fail2ban/jail.local` | 规则错误可能影响登录保护 |
| 防火墙 | `ufw` 或 `firewalld` 规则 | 误删 SSH 端口会断连 |
| 网络参数 | `sysctl`、BBR、TCP 参数 | 错误参数可能影响连通性 |
| 备份目录 | `/etc/vps-optimize/backups` | 占用少量磁盘空间 |

## 操作步骤

### 1. 登录 root 并下载脚本

如果当前不是 root：

```bash
sudo -i
```

下载并运行：

```bash
wget -qO vps.sh https://raw.githubusercontent.com/Chunlion/VPS-Optimize/main/vps.sh && chmod +x vps.sh && ./vps.sh
```

首次运行后会注册快捷命令：

```bash
cy
```

后续都用 `cy` 进入菜单。

### 2. 先做运维预检

进入：

```text
cy -> 1
```

重点看这些结果：

| 检查项 | 你要确认什么 |
|---|---|
| 系统版本 | Debian/Ubuntu 最稳，RHEL 系可用，Alpine 不建议 |
| 磁盘空间 | 至少保留几 GB 可用空间 |
| 内存 | 小内存机器后续可以考虑 ZRAM |
| DNS | 能解析 GitHub、Cloudflare、常用源 |
| 端口 | 先知道当前 SSH 和已有服务端口 |
| 包管理器 | 不要在系统自动更新占用包管理器时继续安装 |

如果预检有异常，先修异常，不要急着装面板或改 SSH。

### 3. 做基础环境初始化

进入：

```text
cy -> 2
```

这一步会安装常用工具、更新基础依赖、设置时间相关组件，并启用基础 BBR。执行后建议检查：

```bash
date -Is
uname -r
sysctl net.ipv4.tcp_congestion_control
```

如果输出里包含 `bbr`，说明基础 BBR 已启用。

### 4. 修改 SSH 端口前先做云安全组准备

不要直接改端口。先在云厂商控制台放行新 SSH 端口，例如 `22222/tcp`。

建议顺序：

1. 在云安全组放行新端口。
2. 保留当前 SSH 窗口。
3. 执行脚本改端口。
4. 新开一个终端测试新端口能登录。
5. 确认成功后，再考虑关闭旧端口。

进入：

```text
cy -> 5
```

高风险确认卡片出现时，只有你确认快照、安全组和当前会话都准备好，才输入大写：

```text
YES
```

验证 SSH：

```bash
ss -lntp | grep ssh
systemctl status ssh --no-pager || systemctl status sshd --no-pager
```

从本地新开终端测试：

```bash
ssh -p 22222 root@你的服务器IP
```

### 5. 添加 SSH 公钥

进入：

```text
cy -> 6
```

粘贴本机公钥。常见公钥位置：

```bash
cat ~/.ssh/id_ed25519.pub
cat ~/.ssh/id_rsa.pub
```

添加后不要立刻禁用密码。先新开终端验证公钥登录成功：

```bash
ssh -p 新端口 root@你的服务器IP
```

确认密钥登录稳定后，再考虑进一步收紧 SSH 登录方式。

### 6. 启用 Fail2ban

进入：

```text
cy -> 7
```

推荐选择安装/配置 Fail2ban。验证：

```bash
systemctl status fail2ban --no-pager
fail2ban-client status sshd 2>/dev/null || fail2ban-client status ssh 2>/dev/null
```

如果服务启动失败，先看日志：

```bash
journalctl -u fail2ban -n 50 --no-pager
```

### 7. 配置系统防火墙

进入：

```text
cy -> 8
```

推荐先选择“启用防火墙 + 自动放行当前公网端口”。脚本会嗅探当前监听端口并追加放行，避免一上来误锁 SSH。

防火墙操作要记住两层：

| 层级 | 在哪里改 | 说明 |
|---|---|---|
| 云安全组 | 云厂商网页控制台 | 决定公网能不能到 VPS |
| 系统防火墙 | `cy -> 8` | 决定进入系统后是否放行 |

验证：

```bash
ufw status numbered 2>/dev/null || firewall-cmd --list-ports
ss -lntp
```

不要删除当前 SSH 端口。关闭防火墙、删除规则都属于高风险操作，必须输入大写 `YES` 才会继续。

### 8. 网络与内核优化

进入：

```text
cy -> 10
```

建议选择策略：

| 目标 | 推荐 |
|---|---|
| 普通 VPS | 使用基础 BBR 即可，不急着换内核 |
| 小内存机器 | 可以考虑 ZRAM / Swap |
| 高并发转发 | 再考虑 TCP 参数调优 |
| 不了解内核差异 | 不要安装/切换优化内核 |

安装/切换内核、清理旧内核都可能影响下次启动。执行前必须确认：

- 已创建快照。
- 云厂商控制台可用。
- 当前系统不是 OpenVZ 老系统。
- 不删除当前正在运行的内核。

### 9. 健康检查

进入：

```text
cy -> 15
```

重点看：

| 项目 | 期望 |
|---|---|
| SSH | 运行中 |
| 防火墙 | 规则符合预期 |
| 失败 systemd 单元数 | 越少越好，最好为 0 |
| 监听端口 | 没有未知公网端口 |
| 证书摘要 | 如尚未部署 HTTPS，可暂时无证书 |

如果要提交 Issue，在健康总览界面按 `d` 生成反馈诊断信息，提交前再检查是否有敏感信息。

### 10. 创建配置备份

进入：

```text
cy -> 16 -> 1
```

查看备份：

```text
cy -> 16 -> 2
```

备份通常包含 SSH、Caddy、Docker、Fail2ban、sysctl 等脚本可管理的关键配置。它不是整机快照，不能替代云厂商快照。

## 验证方法

建议最终执行：

```bash
date -Is
uname -a
ss -lntp
systemctl --failed --no-pager
systemctl status ssh --no-pager || systemctl status sshd --no-pager
systemctl status fail2ban --no-pager
ufw status numbered 2>/dev/null || firewall-cmd --list-ports
```

再进菜单：

```text
cy -> 15
cy -> 16 -> 2
```

## 失败怎么回滚

| 问题 | 优先处理 |
|---|---|
| 新 SSH 端口连不上 | 不要关旧窗口，在旧窗口恢复 SSH 配置或放行端口 |
| 所有 SSH 都断了 | 用云厂商 VNC/救援模式登录，放行端口或恢复配置 |
| 防火墙误删规则 | 控制台登录后重新放行 SSH 端口 |
| Fail2ban 异常封禁 | 临时停止 Fail2ban，确认规则后再启用 |
| 内核切换后无法启动 | 从云厂商控制台选择旧内核或用救援模式恢复 |
| 配置改坏但系统还能登录 | `cy -> 16 -> 3` 从脚本备份回滚 |

## 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 改 SSH 端口前没放行云安全组 | 新端口连不上 | 先放行云安全组，再改系统 |
| 当前 SSH 窗口也关了 | 无法恢复配置 | 保留旧窗口直到新窗口验证成功 |
| 一上来关闭密码登录 | 密钥没配好会失联 | 先验证密钥登录，再逐步收紧 |
| 把云安全组和系统防火墙混为一谈 | 排错方向错误 | 两层都要检查 |
| 没快照就清理内核 | 可能影响启动 | 内核操作前必须快照 |
| 看到警告仍继续 | 问题叠加难排查 | 先修预检异常，再继续部署 |
