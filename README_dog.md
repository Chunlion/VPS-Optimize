# 🐶 端口流量狗

端口流量狗 (Port Traffic Dog) 是一个轻量的 Linux 端口流量管理工具，适合共享节点、端口出租、合租 VPS、临时限流和按端口对账。

它基于 `nftables` 和 `tc` 工作，可以按端口统计流量、设置配额、限制带宽、生成日报趋势，也可以接入 Telegram 机器人，让用户自己查询用量。

> 致谢：本工具核心逻辑基于 [zywe03/realm-xwPF](https://github.com/zywe03/realm-xwPF/blob/main/port-traffic-dog.sh) 增强维护，感谢原作者的开源基础。

## 核心能力

| 能力 | 说明 |
| --- | --- |
| 端口统计 | 支持单端口、多个端口、端口段，例如 `8080`、`8080,8081`、`10000-10050` |
| 流量配额 | 超出配额后自动限制，适合“用完即停” |
| 带宽限速 | 按端口设置速率，例如 `50mbit` |
| 多种口径 | 支持用户单向出站、端口双向、VPS 商家口径 |
| 自动重置 | 可按月固定日期清零端口统计 |
| 日报趋势 | 支持今日、昨日、指定日期、近 7 日趋势 |
| Telegram 查询 | 群内用 `/t 端口`、`/all`、`/trend` 查询 |
| 配置隔离 | 卸载时优先隔离配置，避免误删历史数据 |

## 快速部署

建议使用 Debian / Ubuntu，并以 `root` 身份运行：

```bash
wget -qO dog.sh https://raw.githubusercontent.com/Chunlion/VPS-Optimize/main/dog.sh && chmod +x dog.sh && ./dog.sh
```

安装后通常可以直接输入：

```bash
dog
```

脚本会自动检查并安装依赖：

```text
nftables / iproute2(tc, ss) / jq / bc / curl / cron / conntrack
```

## 推荐使用流程

| 目标 | 推荐操作 |
| --- | --- |
| 第一次使用 | 添加端口监控 → 设置备注 → 选择统计口径 → 观察实时流量 |
| 给用户限流 | 添加端口 → 设置配额或限速 → 测试超限行为 |
| 群内自助查询 | 配置 Telegram 机器人 → 群内测试 `/help` 和 `/all` |

## 添加端口监控

进入：

```text
1. 添加/删除端口监控
1. 添加端口监控
```

支持格式：

```text
8080
8080,8081,8082
10000-10050
8080,10000-10050
```

添加时可以设置：

| 项目 | 建议 |
| --- | --- |
| 流量配额 | 例如 `100G`、`500G`、`1T`，回车表示不限制 |
| 计费口径 | 给用户看通常选单向出站；完整统计选端口双向 |
| 备注 | 写用户、节点、用途，后续排查会轻松很多 |

## 流量口径怎么选

| 口径 | 适合场景 | 说明 |
| --- | --- | --- |
| 用户单向出站 | 给用户看用量 | 只统计 VPS 从被监控端口发给用户的数据 |
| 端口双向 | 看端口完整消耗 | 统计被监控端口入站 + 出站 |
| VPS 商家口径 | 和 VPS 后台大致对账 | 统计整机所有非 `lo` 网卡入站 + 出站 |

注意：脚本里的“VPS 商家口径”读取 `/proc/net/dev`，是本机自启动以来的累计值。VPS 重启后会重新累计；真正月度账单仍以商家后台为准。

## 配额和限速

进入：

```text
2. 配额/限速管理
```

常见需求：

| 需求 | 做法 |
| --- | --- |
| 某端口每月 100G | 设置流量配额 `100G` |
| 用完不断网，只降速 | 设置带宽限制，例如 `10mbit` |
| 给共享端口控速 | 设置端口限速 |
| 只按用户下载量算 | 选择用户单向出站口径 |
| 按端口完整流量算 | 选择端口双向口径 |

配额适合“用完即停”，限速适合“不断网但降速”。两者可以同时使用。

## Telegram 查询

### 1. 获取 Bot Token

1. 在 Telegram 搜索 `@BotFather`。
2. 输入 `/newbot` 创建机器人。
3. 复制返回的 API Token。

### 2. 关闭 Group Privacy

这是 Telegram 机器人不回复的高频原因。

1. 在 `@BotFather` 输入 `/mybots`。
2. 选择你的机器人。
3. 进入 `Bot Settings` -> `Group Privacy`。
4. 点击 `Turn OFF`，确认显示 `Privacy mode is disabled`。

### 3. 在脚本中部署

进入：

```text
7. 通知管理 (Telegram 查询)
1. 部署 Telegram 交互式查询机器人
```

按提示填入：

```text
Bot Token
Chat ID
```

群组 `Chat ID` 通常是以 `-100` 开头的数字。

常用指令：

| 指令 | 作用 |
| --- | --- |
| `/t 端口` | 查询单端口实时流量 |
| `/all` | 查询全部端口实时汇总 |
| `/yday` | 查询昨日日报 |
| `/trend` | 查询近 7 日趋势 |
| `/day YYYY-MM-DD` | 查询指定日期日报 |
| `/help` | 查看帮助 |

示例：

```text
/t 8080
/t 10000-10050
/all
/trend
/day 2026-04-30
```

## 备份、更新和卸载

| 操作 | 入口 | 说明 |
| --- | --- | --- |
| 导出/导入配置 | `4` | 适合迁移机器或重装前保存规则 |
| 更新脚本 | `5` | 检查并热更新脚本 |
| 卸载脚本 | `6` | 清除运行规则和定时任务，配置优先隔离 |

配置文件默认保存到：

```text
/etc/port-traffic-dog/config.json
```

脚本会尽量将配置目录权限收紧为 `700`、配置文件收紧为 `600`，避免 Bot Token 等敏感信息被普通用户读取。

卸载时会清除 `nftables` / `tc` 运行规则和定时任务；配置目录、Telegram 后台服务文件和快捷命令会优先移动到：

```text
/root/port-traffic-dog-quarantine
```

确认隔离内容无用后，再手动清理。

## 常见问题

### 运行脚本提示不是 root

先切换 root：

```bash
sudo -i
```

再重新运行脚本。

### Telegram 机器人不回复

优先检查：

```text
Bot Token 是否填错
Chat ID 是否填错
机器人是否已经加入目标群
Group Privacy 是否已经 Turn OFF
后台服务是否正常
```

查看服务状态：

```bash
systemctl status port-tg-bot --no-pager
```

### 群里查询到的流量和 VPS 后台不一致

这是正常现象。端口流量狗统计的是被监控端口；VPS 后台通常统计整机所有网卡流量。系统更新、Docker 拉镜像、面板、探针、DNS、上游连接都可能算进商家后台。

### 重启 VPS 后规则还在吗

脚本会注册开机自启和定时任务，重启后会尝试恢复监控规则、配额、限速和流量数据。若发现异常，重新进入 `dog` 主菜单通常会触发依赖检查和规则恢复。

### 设置了限速但没生效

检查：

```text
端口是否已经加入监控
网卡是否正常识别
tc 是否存在
是否有其他脚本或面板同时修改 tc 规则
```

### 卸载后想恢复配置

先查看隔离目录：

```text
/root/port-traffic-dog-quarantine
```

如果里面有旧的 `config.json`，可以和新配置对比，或通过导入配置功能恢复。

## 使用建议

- 给用户看流量时，优先看“用户单向出站”。
- 对账 VPS 后台时，参考“VPS 商家口径”。
- 给共享节点设置配额时，建议同时写备注。
- 大量端口段监控前，先从少量端口测试。
- 部署 Telegram 机器人后，先用 `/help` 和 `/all` 测一遍。
- 修改配额、限速、立即清零前，确认端口选择没错。
