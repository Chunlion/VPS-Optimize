# 🐶 端口流量狗 v1.2.6 交互增强版

👉 **一、这是做什么的？**

经常和朋友合租节点，或者给群友分享端口，但总有人跑满流量不知道？

**端口流量狗 (Port Traffic Dog)** 是一个轻量的 Linux 端口流量管理工具，底层基于 `nftables` 和 `tc`：能按端口统计流量、设置月度配额、限制带宽、生成日报趋势，还能通过 Telegram 机器人让群友自己查用量。

> 💡 **致谢说明**：本工具的核心逻辑基于原作者开源项目 [zywe03/realm-xwPF](https://github.com/zywe03/realm-xwPF/blob/main/port-traffic-dog.sh) 进行增强与维护，感谢原作者的优秀底座！

---

👉 **二、核心功能亮点**

- ✅ **端口级统计**：按单端口或端口段统计流量，例如 `8080`、`10000-10050`。
- ✅ **配额断网**：超出流量配额后自动下发限制规则，防止继续跑量。
- ✅ **带宽限速**：按端口设置速率限制，适合共享节点或临时限流。
- ✅ **单双向计费**：可选择只看 VPS 出站，或统计入站 + 出站。
- ✅ **商家口径展示**：主界面展示整机网卡入站、出站、合计流量，方便和 VPS 后台对账。
- ✅ **自动重置**：可设置每月固定日期自动清零端口统计。
- ✅ **日报趋势**：查看昨日、指定日期、近 7 日趋势。
- ✅ **Telegram 查询**：群里发送 `/t 端口`、`/all`、`/trend` 等命令即可查询。
- ✅ **配置隔离**：卸载时配置目录会先隔离保留，避免误删历史数据。

👉 **三、快速部署**

建议使用 Debian / Ubuntu，并以 `root` 身份运行。

```bash
wget -qO dog.sh https://raw.githubusercontent.com/Chunlion/VPS-Optimize/main/dog.sh && chmod +x dog.sh && ./dog.sh
```

安装后通常可以直接输入：

```bash
dog
```

进入管理菜单。

脚本会自动检查并安装所需依赖，例如：

```text
nftables / iproute2(tc, ss) / jq / bc / curl / cron / conntrack
```

👉 **五、添加端口监控**

进入：

```text

1. 添加端口监控
```

支持这些格式：

```text
8080
8080,8081,8082
10000-10050
8080,10000-10050
```

添加时可以顺手设置：

- **流量配额**：例如 `100G`、`500G`、`1T`，回车默认不限制。
- **计费口径**：单向出站或端口双向。
- **备注**：方便区分用户、节点或用途。

👉 **六、三种流量口径怎么理解？**

**用户单向(出站)**：只统计 VPS 从被监控端口发给用户的数据。这个数最适合给用户看，尤其是下载型流量为主的节点。

**端口双向**：统计被监控端口的入站 + 出站。它能反映这个端口本身产生了多少流量，但不等于 VPS 商家后台的整机流量。

**VPS 商家口径**：统计整台机器所有非 `lo` 网卡的入站 + 出站。商家后台通常按整机口径计费，所以系统更新、Docker、面板、探针、上游连接等流量都会被算进去。

> ⚠️ 注意：脚本里的“VPS 商家口径”读取的是 `/proc/net/dev`，属于本机自启动以来累计值。如果 VPS 重启过，它会从系统启动后重新累计；真正月度账单仍以商家后台为准。

👉 **七、配额和限速怎么用？**

进入：

```text
2. 配额/限速管理
```

常见用法：

| 需求 | 建议 |
| --- | --- |
| 给某个端口每月 100G | 设置流量配额 `100G` |
| 给共享端口限速 | 设置带宽限制，例如 `50mbit` |
| 只想按用户实际下载量算 | 选择单向出站口径 |
| 想按端口完整流量算 | 选择双向口径 |

配额适合“用完即停”，限速适合“不断网但降速”。两者可以同时使用。

👉 **八、Telegram 机器人查询**

**步骤 1：获取 Bot Token**

1. 去 Telegram 搜索 `@BotFather`。
2. 输入 `/newbot` 创建机器人。
3. 复制创建成功后返回的 **API Token**。

**步骤 2：关闭 Group Privacy**

默认情况下机器人看不到群里普通消息，这是最容易踩坑的地方。

1. 在 `@BotFather` 输入 `/mybots`。
2. 选择你的机器人。
3. 进入 **Bot Settings** -> **Group Privacy**。
4. 点击 **Turn OFF**，确保显示 Privacy mode is disabled。

**步骤 3：在脚本中部署**

```text
7. 通知管理 (Telegram 查询)
1. 部署 Telegram 交互式查询机器人
```

按提示填入：

```text
Bot Token
Chat ID
```

群组 `Chat ID` 通常是以 `-100` 开头的数字。

**可用 TG 指令：**

```text
/t 端口           查询单端口实时流量
/all              查询全部端口实时汇总
/yday             查询昨日日报
/trend            查询近 7 日趋势
/day YYYY-MM-DD   查询指定日期日报
/help             查看帮助
```

示例：

```text
/t 8080
/t 10000-10050
/all
/trend
/day 2026-04-30
```

👉 **十、配置备份、更新和卸载**

配置菜单：

```text
4. 导出/导入配置
```

适合迁移机器或重装前保存端口规则、配额、限速、通知配置。

更新脚本：

```text
5. 检查并热更新脚本
```

卸载脚本：

```text
6. 卸载脚本
```

卸载会清除 `nftables` / `tc` 规则、定时任务、Telegram 后台服务和快捷命令；配置目录会移动到隔离目录：

```text
/root/port-traffic-dog-quarantine
```

确认隔离内容无用后，再手动清理。

👉 **十一、常见问题**

### 🔐 运行脚本提示不是 root

先切换 root：

```bash
sudo -i
```

再重新运行脚本。

### 📭 Telegram 机器人不回复

优先检查：

- Bot Token 是否填错。
- Chat ID 是否填错。
- 机器人是否已经加入目标群。
- `@BotFather` 里的 Group Privacy 是否已经 Turn OFF。
- 后台服务是否正常：`systemctl status port-tg-bot --no-pager`。

### 📊 群里查询到的流量和 VPS 后台不一致

这是正常现象。端口流量狗统计的是被监控端口；VPS 后台通常统计整机所有网卡流量。系统更新、Docker 拉镜像、面板、探针、DNS、上游连接都可能算进商家后台。

### 🧱 重启 VPS 后规则还在吗？

脚本会注册开机自启和定时任务，重启后会尝试恢复监控规则、配额、限速和流量数据。若发现异常，重新进入 `dog` 主菜单通常会触发依赖检查和规则恢复。

### 🚦 设置了限速但没生效

检查：

- 端口是否已经加入监控。
- 网卡是否正常识别。
- `tc` 是否存在。
- 是否有其他脚本或面板同时修改了 `tc` 规则。

### 🧹 卸载后想恢复配置

先查看隔离目录：

```text
/root/port-traffic-dog-quarantine
```

如果里面有旧的 `config.json`，可以和新配置对比，或通过导入配置功能恢复。

👉 **十二、使用建议**

- ✅ 给用户看流量时，优先看“用户单向(出站)”。
- ✅ 对账 VPS 后台时，参考“VPS 商家口径”。
- ✅ 给共享节点设置配额时，建议同时写备注。
- ✅ 大量端口段监控前，先从少量端口测试。
- ✅ 部署 Telegram 机器人后，先用 `/help` 和 `/all` 测一遍。
- ⚠️ 修改配额、限速、立即清零前，确认端口选择没错。
